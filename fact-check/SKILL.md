# Fact Check - 历史知识点核查

## 工作流

### 1. 提取知识点

读取目标文件，逐条提取所有可验证的历史事实断言，包括：
- 人物身份、关系、事迹
- 事件经过、时间、地点
- 因果关系和历史影响
- 直接引文或史书原文引用
- 数据、数量、年份

每个知识点独立编号，保持原文表述以便对照。忽略主观评论、现代类比和修辞手法（这些不需要验证）。

### 2. 判断地区，加载参考文献

根据知识点涉及的历史地区，加载对应的史书索引：
- 中国历史 → 读取 `references/sources-china.md`
- 欧洲/西方历史 → 读取 `references/sources-europe.md`
- 日韩/东亚历史 → 读取 `references/sources-japan-korea.md`
- 如涉及多个地区，加载所有相关文件

### 3. 分批验证

使用 Agent 工具并行验证知识点（每批 3-5 个），提高效率：
- 对每个知识点用 WebSearch 搜索，交叉比对至少 2 个来源
- 搜索时优先使用知识点中的关键人名、事件名、年份作为搜索词
- 注意区分"学术主流观点"和"民间传说/演义"，在备注中标明

### 4. 追溯原始文献

根据验证结果和参考文献索引，将每个知识点映射到具体的原始史书出处：
- 优先引用原始史书（如《史记·高祖本纪》），而非后世著作
- 尽可能精确到篇/卷/章节（如《资治通鉴·卷一九七》）
- 如果某个知识点来自笔记/杂史而非正史，标注说明
- 如果某个事实在不同史书中记载有出入，注明各方说法

### 5. 判定并输出报告

对每个知识点标注判定结果：
- ✅ **准确** — 与史书记载一致
- ⚠️ **部分准确** — 大体正确但细节有偏差（说明偏差）
- ❌ **有误** — 与史书记载不符（说明正确信息）
- ❓ **存疑** — 史学界有争议或无法从可靠来源确认
- 📖 **演义/传说** — 来自演义小说或民间传说，非正史记载

将报告写入目标文件同目录下的 `fact-check-report.md`，格式如下：

```markdown
# 事实核查报告

**原文件**：`文件名`
**核查日期**：YYYY-MM-DD
**知识点总数**：N | ✅ x | ⚠️ x | ❌ x | ❓ x | 📖 x

---

| # | 原文知识点 | 判定 | 史书出处 | 备注 |
|---|-----------|------|----------|------|
| 1 | ... | ✅ | 《史记·高祖本纪》 | |
| 2 | ... | ⚠️ | 《资治通鉴》卷一九七 | 实际为… |

---

## 详细说明

### ⚠️ 部分准确的知识点

#### #2: [原文摘要]
- **原文表述**：...
- **史书记载**：...
- **偏差说明**：...

### ❌ 有误的知识点
（同上格式）

### ❓ 存疑的知识点
（同上格式）
```

## 注意事项

- 不要验证修辞手法和现代类比（如"古代版大众点评"），只验证历史事实
- 网络搜索结果仅作为线索，最终出处必须追溯到原始史书文献
- 如果某个知识点过于细碎（如某个形容词的准确性），可合并到相关知识点一起验证
- 报告中只展开说明非 ✅ 的知识点，✅ 的只在表格中列出即可
